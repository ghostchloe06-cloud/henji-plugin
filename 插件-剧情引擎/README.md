# Roche 剧情引擎插件 (Story Engine) 二改开发与部署指南

本插件是专为 **Roche**（纯前端、全信任 JS 角色交互平台）设计打造的**剧情分支出线（IF线）引擎**。它能将用户在主对话中积累的人设、记忆和上下文冻结备份，在不污染主线的前提下开辟一个平行的“沙盒宇宙”，并在其中探索多种剧情可能。

---

## 一、 核心功能详解 (对于新用户的介绍)

为了提供具有沉浸感且合乎逻辑的互动，本插件基于以下三大板块构建：

### 1. 「选择」页 —— 时空锚点冻结与分支开启
当用户点击主对话启动引擎时，插件会瞬间捕获该时空节点（时间点 A）的全部信息，并将其固化保存。
*   **冻结快照**：包括：**角色人设、用户特质、长期核心记忆、长期事实记忆**，以及当时的 **主线上下文**。
*   **防止漂移**：后续即使主线对话继续推进到时间点 B、C 甚至修改了人设，本 IF 分支生成时也**强制只读取冻结在时间点 A 的历史快照**，彻底杜绝因时空演进而导致的 OOC（Out of Character）或记忆错乱。
*   **启动表单**：支持自定义设定 IF 分支的“时间背景”、“感情基调”和“高优先级额外指令”。

### 2. 「续约」页 —— 多模态剧情续写与交互
在分支进行中，用户可在右上方随时切换“线上”和“线下”模式：
*   **线上模式 (微信高仿对讲)**：
    *   页面采用经典微信聊天视觉风格（白/绿气泡，极简无阴影）。
    *   **多气泡机制**：AI 会模仿现代人聊天习惯，一次性连发 **至少 3 条** 简短口语化气泡（利用 `[SPLIT]` 符号或换行符自动解析），而不是发出一大段累赘的文本。
*   **线下模式 (纯文学小说叙事)**：
    *   以第三人称（角色名）和第二人称（“你”）撰写连贯、细腻的长文本小说，排版采用优雅的衬线字体引言框。
    *   **禁越权指令**：Prompt 中有严格约束，**严禁 AI 替用户做决定、代行身体动作、神态描写或替用户说台词**，将反应权和探索权完全归还给玩家。
*   **高级双击消息操作菜单 (Edit/Rollback/Delete)**：
    *   双击任意一条消息气泡，即可唤出操作层：
        *   **编辑**：在线修改单条历史发言并即时存盘。
        *   **重回 (回滚重来)**：从当前消息回溯，找到此消息上方最邻近的一条用户指令，**自动切除这之后的所有序列，并立即自动重新呼叫 AI 生成新走向**（若没有历史发言则自动重新初始化）。
        *   **删除**：物理删除单条消息，下方楼层序号自动递减，无需手动校准。
*   **跨段提炼总结栈 (Summaries Stack)**：
    *   在详情中输入起始与结束楼层，即可一键生成不多于 200 字、语气客观严谨的局部剧情总结。
    *   总结将层级堆叠在下方，支持随时**展开、在线编辑修改与删除**。

### 3. 「后台」页 —— 结案归档与记忆沉淀
当 IF 线走向终局，用户可以将其“结案归档”并转入后台管理。
*   **全尺寸视口展示**：100% 高度自适应滚动视图，完整拉取并回看整条 IF 分支的全部对讲和文学轨迹。
*   **注入主记忆**：点击生成总结，利用 AI 以**不多于 200 字的纯客观陈述性文体**提炼本 IF 分支的核心事件，确认后永久写入 Roche 主事实数据库（Facts 记忆卡片中）。
*   **写入防护**：通过同时向 `summaryText` 和 `action` 字段赋相同内容，解决了因宿主 UI 提取偏好导致卡片显示通用 generic 描述的 Bug。

## 二、 二次开发指南 (面向开发者)

### 1. 技术栈与架构
*   **零编译与无 NPM 依赖**：纯 Vanilla HTML5、CSS3 及原生 JavaScript（ES6），适合新手直接修改部署。
*   **闭包安全隔离**：全代码嵌套在 IIFE 自执行函数中。内部的 API 引用（`currentRoche`）、容器节点（`pluginContainer`）和运行态变量皆为局部变量，卸载时会彻底垃圾回收，不污染宿主全局环境。
*   **事件委托 (Event Delegation)**：双击菜单采用了父级事件委托机制：
    ```javascript
    scrollBox.ondblclick = (e) => {
      const block = e.target.closest(".se-message-block");
      if (block) { ... }
    };
    ```
    即使聊天气泡因新发言或回滚重绘、层级繁多，也绝不会出现由于节点替换导致双击监听失效的 Bug。

### 2. 关键定制点（如何修改 Prompt 改变文风）
如果您想调整 AI 的生成格调，可以定位到 `btnTriggerAi.onclick` 代码块：
*   **若想改变微信口吻**：修改 `systemPrompt = ...` 线上模式（`currentIf.mode === "online"`）里的约束，调整 `[SPLIT]` 的示例。
*   **若想改变文学质感**：修改线下模式（`currentIf.mode === "offline"`）中关于第三人称/第二人称、文笔倾向和行为禁令的措辞。
*   **若想修改客观总结格式**：搜索 `【严格铁律限制】`，修改总结的客观文体以及字数约束。

### 3. 颜色风格调整
全插件使用现代浅色护眼色调构建，如需更改配色，可在最下方的 CSS 样式表中，直接修改根选择器 `.roche-plugin-story-engine` 下定义的 CSS 自定义属性（CSS Variables）：
```css
.roche-plugin-story-engine {
  --se-bg: #f8fafc; /* 主背景：柔和 slate 灰白 */
  --se-surface: #ffffff; /* 卡片/面板背景：白 */
  --se-border: #e2e8f0; /* 极浅灰边框 */
  --se-primary: #4f46e5; /* 主色：靛蓝 */
  --se-primary-hover: #4338ca; 
}



三、 部署与新手安装步骤
步骤 1：创建 GitHub 仓库
创建一个公开的 GitHub 仓库，例如命名为 roche-if。
将修正后的 plugin.js 和您替换了占位符的 manifest.json 上传至该仓库根目录。
步骤 2：获取 Raw 链接
找到您仓库中的 manifest.json，点击 GitHub 界面上的 Raw 按钮，复制浏览器地址栏中的链接。其格式应该类似于：

https://raw.githubusercontent.com/<您的用户名>/roche-if/main/manifest.json



步骤 3：在 Roche 中安装
打开 Roche 系统。
进入系统的 插件管理 (Plugins)。
点击 安装远程插件，将您刚刚复制的 manifest.json 的 Raw 链接粘贴进去，点击确认。
在弹出的权限与风险提示页面中点击“同意并安装”。
安装完成后，您就可以在 Roche 的应用列表/侧边栏中开启属于您的平行剧情宇宙了！
